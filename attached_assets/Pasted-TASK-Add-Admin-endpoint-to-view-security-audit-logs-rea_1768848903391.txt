FILES:

- shared/schema.ts already exports `securityAuditLog` table (do not change schema)

REQUIREMENTS:
1) Add endpoint:
   GET /api/admin/security-audit
   - Protected by adminAuth
   - Read-only; NO inserts/updates/deletes
   - Returns latest logs in descending time order

2) Query params (all optional):
   - eventPrefix: string  (example: "admin_hunt_wipe_")
   - eventType: string    (exact match)
   - severity: string     ("info" | "warning" | "critical")
   - userId: string
   - limit: number (default 50; clamp to 1..200)
   - sinceIso: string (ISO date; only return createdAt >= sinceIso)
   - cursorIso: string (ISO date; pagination: return createdAt < cursorIso)

3) Response format:
{
  success: true,
  count: number,
  nextCursorIso: string | null,
  items: Array<{
    id: string,
    userId: string | null,
    eventType: string,
    severity: string,
    details: string | null,
    clientIp: string | null,
    userAgent: string | null,
    createdAt: string
  }>
}

4) Pagination logic:
- Always sort by createdAt DESC.
- If cursorIso provided, filter createdAt < cursorIso.
- nextCursorIso should be the last item’s createdAt (ISO) if items.length === limit, else null.

5) Safety:
- Validate sinceIso/cursorIso parsing; if invalid, return 400 with { error: "INVALID_DATE" }.
- Clamp limit; if invalid number, use default 50.
- Never throw raw errors; return 500 { error: "FAILED_TO_FETCH_AUDIT_LOGS" } and console.error.
