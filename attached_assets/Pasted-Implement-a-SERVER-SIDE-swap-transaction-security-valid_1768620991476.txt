Implement a SERVER-SIDE swap transaction security validator that correctly resolves v0 Address Lookup Table (LUT) accounts (so we stop false “unexpected program” blocks) WITHOUT changing or breaking existing swap flows. Keep changes minimal and additive.

GOAL
- Client-side swapSecurity.ts cannot reliably verify Jupiter v0/LUT txs. Add a server validator that:
  1) Deserializes the base64 VersionedTransaction
  2) Resolves LUT accounts via RPC (connection.getAddressLookupTable)
  3) Extracts the REAL program IDs used by compiled instructions
  4) Enforces an allowlist (Jupiter + Pump + core Solana programs)
  5) Enforces fee payer / signer rules (must be the user)
- Return this validator result inside existing swap API responses so the client can rely on server truth for LUT txs.

CONSTRAINTS
- Do NOT refactor swap logic or change how quotes/swaps are built.
- Do NOT change request/response shapes in a breaking way; only ADD an optional `security` field to swap responses.
- Keep any new files in server/swap/*.
- Use existing `swapConfig` / connection config. If there’s an existing Solana RPC connection in swap code, reuse it.
